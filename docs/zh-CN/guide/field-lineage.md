# 字段血缘

字段血缘功能帮助你理解数据库中字段之间的关联关系，追踪数据的来源和去向。DBX 会从多个维度自动分析字段之间的关联，并标注置信度。

## 关联来源

DBX 通过以下方式检测字段之间的关联关系：

### 外键关系（确定置信度）

基于数据库中已定义的外键约束，这是最可靠的关联来源。例如，`orders.user_id` 通过外键指向 `users.id`，这种关联是确定的。

### 视图引用（高置信度）

分析视图定义中的 SQL 语句，提取视图字段与底层表字段的映射关系。由于视图定义是明确的 SQL 逻辑，这类关联具有较高的置信度。

### 查询历史引用

从你执行过的 SQL 查询历史中，分析 `JOIN` 条件和 `WHERE` 子句，推断字段之间可能存在的关联。

### 同名字段检测

检测不同表中名称相同的字段，提示它们之间可能存在逻辑关联。这类关联的置信度相对较低，需要人工判断。

## 搜索与筛选

你可以通过以下方式快速定位关注的关联：

- **搜索** -- 输入表名或字段名，过滤显示相关的血缘关系
- **按置信度筛选** -- 选择只查看特定置信度级别的关联，例如仅显示"确定"和"高"置信度的结果

## 跳转到关联表

在血缘视图中，点击任意关联的表或字段，可以直接跳转到该表的数据浏览或结构查看页面，方便你快速了解上下游数据。
